Privacy Policy

Last Updated: October 12, 2025

IMPORTANT NOTICE: This is a placeholder privacy policy template intended for development and testing purposes. While it reflects the actual privacy practices of openElara, it should be reviewed by legal counsel before any commercial deployment or public distribution.

1. Overview

openElara is an open-source, local-first AI companion application. This privacy policy explains how the application handles your data. The fundamental principle is simple: Your data stays on your device.

2. Data Collection and Storage

2.1 Local-Only Architecture

openElara is designed with a local-first philosophy:

  • No Cloud Storage: All your data (conversations, files, settings, API keys) is stored exclusively on your local device.
  • No Telemetry: The application does not collect, transmit, or analyze usage data, analytics, crash reports, or any other telemetry.
  • No Account Required: openElara does not require user accounts, authentication, or any form of user identification.
  • No Server Communication: The application does not communicate with any openElara servers because there are no openElara servers.

2.2 Data Stored Locally

The following data is stored on your device in %AppData%\Roaming\openelara\ (Windows) or equivalent OS-specific locations:

  • Conversation History: Chat conversations with AI characters (stored in local ChromaDB vector database)
  • Knowledge Base: Files you ingest for retrieval-augmented generation (stored in local ChromaDB database)
  • Settings: Application preferences, theme choices, selected models, character selection
  • API Keys: Third-party API keys (encrypted using operating system's secure storage)
  • Generated Content: Images, videos, and research outputs you create
  • Custom Personalities/Prompts: Any custom AI personalities or prompt templates you create

3. Third-Party Services

3.1 External API Usage

openElara connects to third-party AI services only when you explicitly configure and use them:

  • Ollama (Local): Runs entirely on your machine with no external communication
  • TogetherAI: Cloud-based LLM provider (if you configure their API key)
  • OpenAI-Compatible APIs: Any compatible service you manually configure
  • Exa.ai: Web research service (if you configure their API key)
  • AI/ML API: Video generation service (if you configure their API key)

Important: When you use these third-party services, your prompts, queries, and generated content are sent to those providers according to their privacy policies. openElara does not control or have visibility into how those services handle your data. Review each provider's privacy policy before using their services.

3.2 API Key Security

API keys for third-party services are:

  • Encrypted using your operating system's secure storage mechanism (Electron safeStorage)
  • Stored only on your local device
  • Never transmitted to openElara (we have no servers to receive them)
  • Only decrypted in memory when making API calls

4. Data Sharing

4.1 No Data Sharing by openElara

openElara does not:

  • Share your data with any third parties
  • Sell your data
  • Analyze your usage patterns
  • Collect aggregate or anonymous statistics
  • Use your data for training or research

4.2 User-Initiated Sharing

You control all data sharing:

  • If you use third-party AI APIs, you are sharing data with those providers under their terms
  • If you use web research features (Exa, Scrapy), you are requesting data from those services
  • If you export or save content, you control where it goes

5. Data Retention and Deletion

5.1 Your Control

You have complete control over data retention:

  • Conversation History: Delete individual turns or clear all history via "View & Manage Memories"
  • Knowledge Base: Remove individual files or clear entire knowledge base via "View & Manage KB"
  • Generated Content: Delete files from Output folders at any time
  • Settings: Reset to defaults or manually edit configuration files
  • Complete Removal: Uninstall the app and delete %AppData%\Roaming\openelara\ to remove all data

5.2 Automatic Deletion

openElara does not automatically delete any of your data. Data persists until you explicitly remove it.

6. Security

6.1 Local Security Measures

  • Encrypted API Keys: Third-party API keys are encrypted using OS-level secure storage
  • Content Security Policy: Strict CSP prevents unauthorized script execution
  • Context Isolation: Renderer processes cannot directly access Node.js APIs
  • No Remote Code Execution: Application does not download or execute remote code

6.2 Your Responsibility

Because openElara is a local application, security depends on:

  • Securing your device with passwords, encryption, etc.
  • Keeping your operating system and openElara updated
  • Safeguarding your third-party API keys
  • Backing up your data if needed (openElara does not provide backup services)

7. Open Source Transparency

openElara is fully open source. You can:

  • Review all source code to verify privacy claims
  • Audit network requests to confirm no unauthorized data transmission
  • Build from source to ensure no modifications
  • Fork and modify for your own use under the license terms

8. Children's Privacy

openElara is not directed at children under 13. The application does not knowingly collect data from children. Because the app is local-only and requires manual configuration of API keys, parental supervision is recommended for use by minors.

9. International Users

Because openElara is a local-only application with no servers, there is no cross-border data transfer by the application itself. However, if you use third-party APIs based in other countries, review their data handling practices.

10. Changes to This Policy

Updates to this privacy policy will be:

  • Included in new versions of the application
  • Documented in the app's changelog
  • Posted in the GitHub repository

Continued use of openElara after updates constitutes acceptance of the revised policy.

11. Your Rights

Because all data is stored locally on your device, you have complete rights to:

  • Access: All data is in accessible formats (JSON, markdown, SQLite/ChromaDB)
  • Modify: Edit configuration files or database contents directly
  • Delete: Remove any or all data at any time
  • Export: Copy data folders to other locations or devices
  • Port: Migrate data to other applications that support the formats
